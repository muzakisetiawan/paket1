DNS Server (BIND9)

Domain: lab-smk.xyz ‚Üí 192.168.30.10

DNS berfungsi menerjemahkan nama domain menjadi IP. Di sini kita akan menggunakan domain lokal:

domain
lab-smk.xyz

Install BIND9

bash
sudo apt install bind9 -y

Konfigurasi Zone

bash
sudo nano /etc/bind/named.conf.local

Tambahkan:

config
zone "lab-smk.xyz" {
    type master;
    file "/etc/bind/db.lab-smk";
};

Buat File Zone

bash
sudo cp /etc/bind/db.local /etc/bind/db.lab-smk
sudo nano /etc/bind/db.lab-smk

Isi:

zone file
$TTL 300
@   IN  SOA lab-smk.xyz. root.lab-smk.xyz. (
        2026012201
        3600
        900
        604800
        300 )

@       IN  NS  ns.lab-smk.xyz.
@       IN  A   192.168.30.10
ns      IN  A   192.168.30.10
www     IN  A   192.168.30.10
monitor IN  A   192.168.30.10

Restart DNS

bash
sudo systemctl restart bind9

Pengetesan dari Windows

Atur DNS Windows ke 192.168.30.10, lalu:

cmd
ping lab-smk.xyz
ping www.lab-smk.xyz
ping monitor.lab-smk.xyz

HTTPS Self-Signed Certificate

Enkripsi koneksi dengan SSL/TLS

Pada bagian ini kita akan mengaktifkan HTTPS agar website dapat diakses menggunakan https:// dan koneksi menjadi terenkripsi.

üîí INFO: IP Address: 192.168.30.10  |  Domain: lab-smk.xyz

1. Install OpenSSL

bash
sudo apt update
sudo apt install openssl -y

2. Aktifkan Modul SSL Apache

bash
sudo a2enmod ssl

3. Buat Folder SSL

bash
sudo mkdir /etc/apache2/ssl

4. Buat Sertifikat SSL Self-Signed

bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
-keyout /etc/apache2/ssl/apache.key \
-out /etc/apache2/ssl/apache.crt

Saat muncul pertanyaan, isi seperti berikut:

input
Country Name : ID
State        : Jawa Barat
City         : Bandung
Organization : SMK Insan Mandiri
Unit         : TKJ
Common Name  : lab-smk.xyz
Email        : admin@lab-smk.xyz
‚ö†Ô∏è PENTING: Bagian Common Name sangat penting. Isi dengan domain server: lab-smk.xyz

5. Konfigurasi HTTPS Apache

bash
sudo nano /etc/apache2/sites-available/ssl.conf

Isi:

apache config
<VirtualHost *:443>
  ServerAdmin admin@lab-smk.xyz
  DocumentRoot /var/www/html
  SSLEngine on
  SSLCertificateFile /etc/apache2/ssl/apache.crt
  SSLCertificateKeyFile /etc/apache2/ssl/apache.key
  <Directory /var/www/html>
    AllowOverride All
  </Directory>
</VirtualHost>

Save: CTRL+O ‚Üí ENTER ‚Üí CTRL+X

6. Aktifkan Site HTTPS

bash
sudo a2ensite ssl.conf

7. Redirect HTTP ‚Üí HTTPS

bash
sudo nano /etc/apache2/sites-available/000-default.conf

Ubah menjadi:

apache config
<VirtualHost *:80>
  ServerName lab-smk.xyz
  Redirect / https://lab-smk.xyz/
</VirtualHost>

8. Restart Apache

bash
sudo systemctl restart apache2

9. Test di Browser

url
https://lab-smk.xyz
https://192.168.30.10
‚ÑπÔ∏è INFO: Jika muncul peringatan keamanan: klik Advanced ‚Üí Accept Risk ‚Üí Continue

10. Cek Status Apache

bash
sudo systemctl status apache2
‚ö†Ô∏è CATATAN: Karena menggunakan Self-Signed, browser mungkin menampilkan peringatan. Ini normal dan bukan error.

Zabbix Server

Monitoring server ‚Äî Versi aman & pasti jalan

‚ö†Ô∏è CATATAN: Bagian ini memakai cara manual schema database karena pada beberapa sistem folder sql bawaan Zabbix tidak tersedia.

Update Sistem

bash
sudo apt update && sudo apt upgrade -y

Install MariaDB

bash
sudo apt install mariadb-server -y

Secure MariaDB

bash
sudo mysql_secure_installation

Gunakan jawaban:

input
Switch to unix_socket authentication ‚Üí Y
Set root password ‚Üí N
Remove anonymous users ‚Üí Y
Disallow root login remotely ‚Üí Y
Remove test database ‚Üí Y
Reload privilege tables ‚Üí Y

Buat Database Zabbix

bash
sudo mysql
sql
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
CREATE USER 'zabbix'@'localhost' IDENTIFIED BY '1';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
FLUSH PRIVILEGES;
EXIT;
‚ÑπÔ∏è INFO: Password database adalah 1. Jangan memasukkan password Ubuntu saat diminta password MySQL.

Install Repository Zabbix

bash
wget https://repo.zabbix.com/zabbix/7.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo dpkg -i zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo apt update

Install Zabbix

bash
sudo apt install zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-agent -y

bash

Firewall MikroTik

Filter Rules sesuai soal

Pada bagian ini kita akan mengatur Firewall Filter Rules di MikroTik untuk mengamankan jaringan antar VLAN serta akses ke router.

‚ÑπÔ∏è INFO: Masuk ke MikroTik melalui: IP ‚Üí Firewall ‚Üí Filter Rules

a. Blok Akses VLAN 20 ke VLAN 10

Rule ini harus berada DI ATAS rule allow agar benar-benar bekerja.

mikrotik
Chain        : forward
Src. Address : 192.168.20.0/24
Dst. Address : 192.168.10.0/24
Action       : drop
Comment      : BLOCK VLAN20 to VLAN10

b. Izinkan VLAN 30 ke VLAN 10

mikrotik
Chain        : forward
Src. Address : 192.168.30.0/24
Dst. Address : 192.168.10.0/24
Action       : accept
Comment      : ALLOW VLAN30 to VLAN10
‚ö†Ô∏è PENTING: Pastikan rule ini berada DI ATAS rule DROP VLAN20, karena MikroTik membaca firewall dari atas ke bawah.

c. Pengamanan Brute-Force (SSH & Winbox)

Rule 1 ‚Äî Deteksi

mikrotik
Chain              : input
Protocol           : tcp
Dst Port           : 22,8291
Connection State   : new
Action             : add-src-to-address-list
Address List       : brute-force
Address List Time  : 1d

Rule 2 ‚Äî Blok

mikrotik
Chain            : input
Src Address List : brute-force
Action           : drop
Comment          : BLOCK BRUTE FORCE
‚ÑπÔ∏è INFO: IP yang mencoba login berulang kali akan otomatis diblokir selama 1 hari.

d. Logging Aktivitas Router

mikrotik
Chain      : input
Action     : log
Log Prefix : ROUTER-ACCESS
‚ö†Ô∏è CATATAN: Jangan menambahkan terlalu banyak rule log karena dapat membuat router menjadi berat.

e. Pembatasan Akses Router

Allow VLAN 10

mikrotik
Chain       : input
Src Address : 192.168.10.0/24
Protocol   : tcp
Dst Port   : 22,8291
Action     : accept

Allow VLAN 30

mikrotik
Chain       : input
Src Address : 192.168.30.0/24
Protocol   : tcp
Dst Port   : 22,8291
Action     : accept

Drop Selain Itu

mikrotik
Chain    : input
Protocol : tcp
Dst Port : 22,8291
Action   : drop
Comment  : DROP OTHER ACCESS
‚ÑπÔ∏è INFO: Hanya perangkat dari VLAN 10 dan VLAN 30 yang bisa mengakses router menggunakan SSH dan Winbox.

Konfigurasi Netplan

Opsional ‚Äî Mengatur IP Address Ubuntu Server

Bagian ini bersifat opsional, digunakan untuk mengatur IP Address pada Ubuntu Server menggunakan Netplan.

‚ÑπÔ∏è INFO: File konfigurasi Netplan biasanya berada di: /etc/netplan/50-*.yaml

1. Cek Nama Interface

bash
ip a

2. DHCP ‚Üí Static IP

bash
sudo nano /etc/netplan/50-cloud-init.yaml
yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses:
        - 192.168.30.10/24
      gateway4: 192.168.30.1
      nameservers:
        addresses:
          - 192.168.30.10
          - 8.8.8.8
bash
sudo netplan apply
‚ÑπÔ∏è INFO: Jika koneksi terputus, buka kembali VirtualBox untuk memperbaiki konfigurasi.

3. Static IP ‚Üí DHCP

yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: yes
bash
sudo netplan apply

4. Cek IP Address

bash
ip a
